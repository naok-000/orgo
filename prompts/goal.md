以下を達成するアプリケーションを作成してほしい．

## 目的

k1LoW/mo のUXと org-roam/org-roam-ui の機能を両立させたアプリケーションを作成すること．

## 要件

- org-roam-ui の機能
  - org-roam のファイルをブラウザ上で閲覧できる
    - org-roam のリンクをたどれる
  - org-roam のリンク構造がグラフとして可視化される
    - org-roam-ui の機能を参考に..

- k1LoW/mo のUX
  - ファイルをタブ的に開き，切り替えられる
  - その他， k1LoW/mo のUXを参考にした機能を実装すること
    - これは，k1LoW/mo の実装をそのまま使うこと，そのままのUIを使うことを意味するわけではない．あくまでUXを参考にし，必要・不要な機能は適宜取捨選択すること．

- 配布
  - GitHub のみで公開．Melpaやbrewなどのパッケージマネージャーでの配布はしない．
  - emacs の use-package でインストールできるのが望ましい
    - これが難しい，より良い方法があるようであれば，それを採用する．

- 言語
  - 問わない

- 開発環境
  - nix-flakes と direnv を用いた開発環境を構築すること．

## 実装のすすめ方

- あなたはオーケストレータとして機能する．実装はSonnet5のsubagentに任せる．
- 調査の必要がある場合はCodexに調査を依頼するとよいだろう． `codex`コマンドから利用できるだろう．(方法は調べて)
- コミット・プッシュは適宜行う．

## レビュー

- 第三者目線でのコードレビューを積極的に行うこと．
  - codexを用いるとよいだろう．
- その他，実装判断に迷ったらcodexに相談するとよいだろう．

- goalを達成したかどうかは，codexにこのファイルを渡して判断してもらうこと．

## テスト

- 自動テストを用いてQuality Assuranceを行うこと．
